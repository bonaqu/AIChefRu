# EasyChefAI RU — статическое веб‑приложение

Полностью русскоязычный аналог приложения «рецепты из любых ингредиентов», без сервера. Работает как обычная веб‑страница (статический сайт), легко публикуется на GitLab Pages, адаптирован под телефон и ПК. Есть офлайн‑режим (PWA) после первого открытия.

## Возможности
- Ввод ингредиентов: чек‑лист по категориям + ручной ввод с поддержкой синонимов
- Фильтры: диеты (веган, вегетарианец, без лактозы, без глютена, без орехов, халяль)
- Фильтры по технике: плита, духовка, микроволновка, блендер, мультиварка, «без плиты»
- Опции: порции, время, острота, одной посудой, «для детей», автозамены под диету
- Генерация рецепта локально (имитация ИИ) по шаблонам + гибкая логика подбора
- «Живой ИИ»: анимация «печатает», история последних рецептов, избранное
- Экспорт: копирование, .txt, печать/PDF, системное «Поделиться» (если поддерживается)
- Адаптивная, современная верстка, светлая/темная тема
- PWA: иконка, манифест, сервис‑воркер (кэширует файлы для работы офлайн)

## Структура проекта
```
index.html           — основная страница
styles.css           — стили
data.js              — данные: ингредиенты, диеты, шаблоны
app.js               — логика: UI, генерация рецепта, офлайн и т.д.
manifest.webmanifest — PWA‑манифест
service-worker.js    — сервис‑воркер для офлайн‑режима
favicon.svg          — иконка сайта
README.md            — это руководство
.gitlab-ci.yml       — конфигурация публикации на GitLab Pages
```

## Как запустить локально (Windows, WebStorm)
1. Скачайте проект (или склонируйте репозиторий) в любую папку.
2. Откройте папку проекта в WebStorm.
3. Встроенный сервер WebStorm:
   - Найдите `index.html` в проводнике проекта, кликните правой кнопкой → Open in Browser → выберите браузер.
   - WebStorm обычно открывает по адресу вида `http://localhost:63342/...` — сервис‑воркер будет работать. Если открылся как `file:///`, включите встроенный сервер: File → Settings → Build, Execution, Deployment → Debugger → отметьте «Allow unsigned requests» и используйте Open in Browser снова.
4. Для телефона: отправьте себе ссылку `http://<ваш_ПК>:63342/...` (ПК и телефон должны быть в одной сети) — либо просто протестируйте в мобильной вкладке DevTools.

Советы:
- После первого открытия сайт кэшируется и работает офлайн (PWA). Для обновления кэша просто перезагрузите страницу (или закройте/откройте вкладку).
- Если что‑то не обновляется — очистите кэш сайта в настройках браузера.

## Как опубликовать на GitLab Pages
1. Создайте новый проект на GitLab и загрузите туда файлы из этой папки.
2. Убедитесь, что в корне репозитория есть файл `.gitlab-ci.yml` (он уже в проекте).
3. Сделайте коммит и push в ветку `main` или `master`.
4. Зайдите в CI/CD → Pipelines и дождитесь выполнения job `pages`.
5. После завершения у проекта появится адрес вида `https://<username>.gitlab.io/<project>/` — это и есть ваш сайт.

Ничего собирать не нужно — это статические файлы.

## Использование
1. Шаг 1: выберите ингредиенты из чек‑листа или введите вручную (через запятую). Выбранные ингредиенты отображаются чипсами.
2. Шаг 2: отметьте диеты/технику, задайте порции, время, остроту. При необходимости включите автозамены — приложение корректно подменит запрещённые ингредиенты (например, сыр → тофу для веганов) и сообщит об этом.
3. Шаг 3: нажмите «Сгенерировать рецепт». Можно запросить «Ещё вариант». Сохраните рецепт, экспортируйте, распечатайте или поделитесь.

## Приватность и офлайн
- Все данные и «ИИ‑логика» работают локально в вашем браузере. Никаких серверов.
- История и избранное сохраняются в `localStorage` браузера на вашем устройстве.

## Частые вопросы
- Можно ли подключить реальный OpenAI API? Да, но это потребует серверной прокладки или скрытия ключа — GitLab Pages публикует статический сайт, поэтому ключ нельзя хранить в клиентском JS. В этом проекте используется локальная генерация.
- Почему иногда рецепт «общий»? Это намеренно — логика генерации гибкая и подставляет ваши ингредиенты, но без избыточной «магии», чтобы всё было воспроизводимо.
